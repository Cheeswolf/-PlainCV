# 白简历 PlainCV

白简历 PlainCV 是一个固定模板的在线简历制作工具。用户通过结构化表单填写内容，在桌面端实时查看右侧 A4 预览，并通过浏览器打印功能导出可选择、可复制文字的 PDF。

项目不调用 AI API，不上传简历数据，也不评价、审核或自动修改用户填写的内容。

## 功能

- 固定简历模块：个人信息、教育背景、实习经历、项目经历、个人优势
- 支持添加和删除多段教育、实习及项目经历
- 桌面端左侧编辑、右侧吸顶实时预览
- 移动端编辑与预览切换
- 使用 `localStorage` 自动保存并恢复数据
- 损坏的本地数据自动回退为空白简历
- 实习工作内容和项目描述支持有限加粗
- 三类可查看、展开和复制的 AI 辅助提示词
- A4 白底黑字固定模板
- 支持自然多页和浏览器打印导出 PDF
- 打印时仅保留简历，导出文字可选择和复制

## 产品原则

- 不限制输入字数、行数、经历数量或简历页数
- 不检查内容质量、量化结果或岗位匹配程度
- 不自动改写用户内容
- 不调用 AI API，不自动读取或提交简历内容
- 不使用后端、数据库或用户登录
- 简历模板保持单栏，用户不能修改字体、字号、颜色和模块顺序
- 仅支持受控加粗，不支持任意 HTML 或完整富文本排版

## 简历字段

项目只提供以下字段：

- 个人信息：姓名、手机号、邮箱
- 教育背景：学校、专业、学历、入学时间、毕业时间、主修课程、GPA 或专业排名
- 实习经历：公司名称、岗位名称、开始时间、结束时间、工作内容
- 项目经历：项目名称、项目角色、开始时间、结束时间、项目描述
- 个人优势：自由填写的多行文本

## 有限加粗

实习工作内容和项目描述支持加粗选中的文字。编辑器使用受控 Markdown 标记保存加粗信息：

```text
**需求分析**：整理业务需求并跟踪实施进度。
```

预览和 PDF 会将“需求分析”渲染为粗体。平台只解析成对的 `**文字**`，不成对的标记按原始文本显示，也不会执行用户输入的 HTML。

## AI 辅助提示词

工具箱提供三类固定提示词：

1. 实习经历优化
2. 项目经历优化
3. 个人优势生成或优化

提示词需要由用户主动复制到自己选择的大模型平台。PlainCV 不会调用模型接口，也不会自动将简历内容发送到外部服务。

## 本地运行

建议使用 Node.js 20 或更高版本。

```bash
git clone https://github.com/Cheeswolf/-PlainCV.git PlainCV
cd PlainCV
npm install
npm run dev
```

打开终端显示的本地地址，通常为：

```text
http://localhost:3000
```

如果 `3000` 端口已被占用，Next.js 会显示实际使用的端口。若同一项目已有开发服务器运行，需要先停止旧进程再重新启动。

Windows PowerShell 中也可以使用：

```powershell
npm.cmd install
npm.cmd run dev
```

## 可用命令

```bash
npm run dev        # 启动开发服务器
npm run lint       # 执行 ESLint
npm run test       # 执行自动化测试
npm run typecheck  # 执行严格 TypeScript 检查
npm run build      # 创建生产构建
npm run start      # 启动生产构建
```

## 导出 PDF

1. 填写简历并查看实时预览。
2. 点击“导出 PDF”。
3. 在浏览器打印窗口中选择“另存为 PDF”。
4. 建议选择 A4 纸张并关闭浏览器附加的页眉和页脚。

打印样式会隐藏导航栏、编辑表单、提示词和操作按钮，仅输出真实尺寸的 A4 简历。内容超过一页时会自然分页，不会被固定高度裁切。

## 数据与隐私

简历数据只保存在当前浏览器的 `localStorage` 中：

- 页面刷新或关闭浏览器后仍可恢复
- 点击“清空内容”并确认后会删除本地数据
- 更换浏览器、设备或清理浏览器数据后无法自动恢复
- 项目不提供云端同步、账号系统或后端备份

## 项目结构

```text
src/
├─ app/          # Next.js 页面入口和全局样式
├─ components/   # 编辑器、表单、预览、提示词和打印组件
├─ config/       # AI 辅助提示词配置
├─ data/         # 默认简历数据
├─ hooks/        # localStorage 保存与恢复逻辑
├─ test/         # 测试环境配置
└─ types/        # 简历类型与 Zod 数据结构

docs/
├─ PRODUCT_REQUIREMENTS.md  # 现行产品需求
└─ DEVELOPMENT_PLAN.md      # 开发进度
```

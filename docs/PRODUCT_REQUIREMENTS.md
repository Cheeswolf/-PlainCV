# 白简历 PlainCV 产品方案 V1.1（现行）

## 一、产品定位

这是一个固定模板的在线简历制作工具。

用户可以：

1. 按照固定字段填写个人简历内容。
2. 使用平台提供的提示词，前往免费AI大模型生成或优化简历内容。
3. 将生成后的内容粘贴回对应字段。
4. 实时预览白底黑字的标准简历。
5. 导出为PDF文件。

平台不调用AI API，不承担简历内容的生成、审核和判断，只提供内容填写工具、固定简历模板和辅助提示词。

## 二、核心产品原则

### 1. 内容完全由用户负责

平台不判断用户填写的内容是否：

* 太长
* 太短
* 缺乏数据
* 缺乏结果
* 表达空泛
* 与岗位不匹配
* 超过一页

平台不对用户填写内容进行限制、评分或修改。

### 2. 平台只提供辅助提示词

用户可以自行将提示词复制到：

* ChatGPT
* DeepSeek
* 豆包
* 通义千问
* Kimi
* 其他大模型平台

用户生成内容后，自行决定是否使用，并将最终内容粘贴回简历编辑器。

### 3. 固定模板，只提供有限加粗

简历采用统一的白底黑字模板。

用户可以在实习工作内容和项目描述中使用平台提供的加粗功能突出关键信息，但不能修改：

* 字体
* 字号
* 颜色
* 模块位置
* 简历模板布局
* 分割线样式
* 简历整体结构

加粗内容使用受控的 `**文字**` 语法保存和渲染，不支持任意HTML，也不提供斜体、下划线、字号、颜色、对齐或其他自由排版功能。这样既能突出四字概括等关键信息，又能保证导出的简历格式稳定、整洁。

### 4. 不限制简历页数

简历可以自然生成一页、两页或更多页面。

平台不提示简历超过一页，也不强制用户压缩内容。

应届生、实习生、社会招聘用户都可以按照自己的实际经历决定简历长度。

# 三、页面结构

宽度大于 `900px` 的桌面端采用左侧编辑、右侧实时预览的工作台布局，让用户输入时能够持续看到简历效果。右侧预览采用吸顶展示，内容超过可视区域时可以在预览区域内滚动。

宽度不超过 `900px` 的移动端不使用狭窄双栏，提供“编辑”和“预览”切换，默认显示编辑区域。

左右布局只用于网页编辑工作台，最终简历模板始终是单栏纵向排版。

页面整体结构为：

```text
顶部导航栏

桌面端：
左侧：简历内容填写区域、AI辅助提示词区域
右侧：简历实时预览区域

移动端：
编辑/预览切换
编辑视图：简历内容填写区域、AI辅助提示词区域
预览视图：简历实时预览区域

页面操作区域：导出PDF按钮
```

## 1. 顶部导航栏

包含：

* 产品名称
* 保存状态
* 支持作者（仅在官方托管部署启用时显示）
* 清空内容
* 导出PDF

例如：

```text
白简历 PlainCV

已自动保存              清空内容    导出PDF
```

## 2. 简历填写区域

采用纵向排列。

```text
个人信息
教育背景
实习经历
项目经历
个人优势
```

每个模块使用简单的白色表单区域，不使用复杂卡片、渐变或装饰。

实习工作内容和项目描述的多行文本框上方提供“加粗”按钮。用户选中文字后点击按钮，平台在原始文本中插入 `**` 标记；未选择文字时不修改内容。

## 3. AI辅助提示词区域

位于桌面端左侧编辑区域或移动端编辑视图中，并位于简历填写区域之后。

只提供三类提示词：

1. 实习经历优化
2. 项目经历优化
3. 个人优势生成或优化

每个提示词提供：

* 提示词名称
* 简短使用说明
* 查看提示词
* 一键复制

## 4. 简历预览区域

桌面端位于编辑区域右侧并保持可见，必要时按屏幕空间缩放展示并允许区域内滚动；移动端位于预览视图中。

使用白色纸张样式展示最终简历，按照从上到下的方式排版。

简历预览可以自然延伸为多页，不设置一页高度限制。

屏幕缩放只影响网页中的预览尺寸，打印时必须恢复真实A4尺寸。

# 四、固定简历字段

## 1. 个人信息

只包含：

* 姓名
* 手机号
* 邮箱

不添加其他字段。

简历展示形式：

```text
张三

138xxxxxxxx ｜ example@email.com
```

---

## 2. 教育背景

字段只包含：

* 学校
* 专业
* 学历
* 入学时间
* 毕业时间
* 主修课程，选填
* GPA或专业排名，选填

允许用户添加多段教育经历。

简历展示形式：

```text
教育背景

齐鲁工业大学                    2024.09—2028.06
金融科技｜本科
主修课程：统计学、金融学、Python程序设计
专业排名：前20%
```

没有填写的选填字段不显示。

---

## 3. 实习经历

字段只包含：

* 公司名称
* 岗位名称
* 开始时间
* 结束时间
* 工作内容

允许用户添加多段实习经历。

工作内容使用多行文本框，由用户自由填写。

工作内容支持有限加粗。平台只解析 `**文字**` 为粗体，其余文本和换行按照用户原始输入展示。

平台不限制：

* 字数
* 行数
* 内容结构
* 要点数量
* 表达方式

简历展示形式：

```text
实习经历

字节跳动        2024.12—2025.12
实习生

参与需求文档整理、功能测试、Excel数据处理和任务跟踪。
协助整理技术资料、异常样本和项目会议记录。
```

---

## 4. 项目经历

字段只包含：

* 项目名称
* 项目角色，选填
* 开始时间，选填
* 结束时间，选填
* 项目描述

允许用户添加多个项目。

项目描述使用多行文本框，由用户自由填写。

项目描述支持有限加粗。平台只解析 `**文字**` 为粗体，其余文本和换行按照用户原始输入展示。

平台不限制：

* 字数
* 行数
* 描述结构
* 项目数量
* 内容表达方式

简历展示形式：

```text
项目经历

AI简历初筛工作流                2026.03—2026.05
项目角色：工作流开发与测试

使用Coze搭建简历解析、字段校验、岗位匹配和结果输出流程。
参与工作流测试、问题记录和后续迭代。
```

没有填写项目角色或时间时，对应内容不显示。

---

## 5. 个人优势

只有一个多行文本输入框。

用户可以自由填写：

* 一段文字
* 多段文字
* 多条内容

平台不限制内容格式和字数。

简历展示时按照用户输入的换行方式显示。

# 五、简历模板设计

## 1. 视觉风格

采用标准企业简历风格：

* 白色背景
* 黑色文字
* 深灰色辅助文字
* 从上到下排列
* 不使用侧边栏
* 不使用头像
* 不使用图标
* 不使用彩色模块
* 不使用技能进度条
* 不使用复杂图形
* 不使用双栏简历模板结构

## 2. 模块顺序

固定为：

```text
姓名
手机号和邮箱

教育背景

实习经历

项目经历

个人优势
```

用户不能调整模块顺序。

## 3. 多页处理

简历内容超过一页时自然分页。

需要避免：

* 一个模块标题单独出现在上一页底部
* 公司名称和工作内容被不合理拆开
* 项目名称和项目描述被不合理拆开

可以通过打印CSS中的分页规则尽量保证完整性。

不设置：

```css
height: 297mm;
overflow: hidden;
```

而是使用：

```css
width: 210mm;
min-height: 297mm;
```

让简历内容自然向下延伸。

# 六-A、支持作者功能

“支持作者”属于官方托管版本的可选功能，不属于开源本地版本的默认能力。

* 开源仓库和本地启动时默认关闭，不显示入口和二维码。
* 仅当官方托管部署通过服务端环境变量显式启用，并同时配置微信与支付宝收款码后，顶部导航栏才显示“支持作者”。
* 点击入口后使用弹窗展示微信和支付宝两个二维码、收款人核对提示及自愿支持说明。
* 不预设金额，不把支持行为与任何产品功能绑定，不创建订单，不判断或伪造支付成功状态。
* 二维码文件不得提交到公开仓库，也不得放入 `public` 等公开源码目录；由服务端路由读取私密环境变量并返回图片字节。
* 公共配置接口只能返回是否启用、渠道名称、收款人提示和图片路由，不得返回二维码 Base64 原文。
* 二维码向访客显示后仍可被浏览器获取或截图；本要求只保证二维码不作为公开仓库资源分发。
* 弹窗支持键盘关闭、清晰的按钮标签和移动端基础适配。
* 打印时隐藏入口与弹窗，只显示简历。
* 本功能不使用支付 API、后端数据库或支付结果回调。未来微信小程序支付另行设计，不在本阶段实现。

# 七、数据保存

第一版不需要用户注册，也不需要后端数据库。

用户填写的内容保存在当前浏览器中。

使用：

```text
localStorage
```

实现：

* 自动保存
* 页面刷新后恢复
* 关闭浏览器后再次打开仍可恢复
* 用户可以主动清空数据

清空数据前需要弹出确认提示。

# 八、PDF导出

第一版使用浏览器打印功能导出PDF。

用户点击“导出PDF”后执行：

```javascript
window.print();
```

打印时：

* 隐藏顶部导航栏
* 隐藏表单编辑区域
* 隐藏提示词区域
* 只显示简历内容
* 使用A4纸张宽度
* 支持自然分页
* 保留可以复制的文字

用户在浏览器打印窗口中选择：

```text
另存为PDF
```

# 九、技术方案

## 前端框架

使用：

```text
Next.js + React + TypeScript
```

它们分别负责：

* Next.js：搭建整个网页项目
* React：处理表单和实时预览
* TypeScript：减少字段和数据结构错误

## 表单处理

使用：

```text
React Hook Form
```

负责管理用户填写的内容。

## 基础数据校验

可以使用：

```text
Zod
```

但只进行必要的技术校验，例如：

* 数据是不是字符串
* 某个经历是否有唯一ID
* 本地数据格式是否损坏

不对简历内容进行质量判断。

不检查内容长短，不检查是否量化，不检查是否有结果。

## 数据保存

使用：

```text
localStorage
```

第一版不需要后端和数据库。

## 官方部署支持作者配置

允许使用 Next.js Route Handler 读取服务端私密环境变量，并仅用于返回支持作者功能的公开配置与二维码图片字节。该路由不是支付后端，不处理订单、金额、支付状态或用户简历数据。

所有支持作者环境变量均为服务端变量，不使用 `NEXT_PUBLIC_` 前缀。二维码数据不得进入客户端构建产物或公开仓库。

## 页面样式

使用：

```text
CSS Modules
```

第一版不需要引入复杂组件库。

## PDF导出

使用：

```text
浏览器打印功能 + CSS打印样式
```

第一版不使用收费PDF服务，也不将简历转换成图片。

# 十、第一版数据结构

```typescript
interface ResumeData {
  personalInfo: PersonalInfo;
  education: EducationItem[];
  internships: InternshipItem[];
  projects: ProjectItem[];
  advantages: string;
}

interface PersonalInfo {
  name: string;
  phone: string;
  email: string;
}

interface EducationItem {
  id: string;
  school: string;
  major: string;
  degree: string;
  startDate: string;
  endDate: string;
  courses?: string;
  gpaOrRanking?: string;
}

interface InternshipItem {
  id: string;
  company: string;
  role: string;
  startDate: string;
  endDate: string;
  content: string;
}

interface ProjectItem {
  id: string;
  name: string;
  projectRole?: string;
  startDate?: string;
  endDate?: string;
  description: string;
}
```

平台保存用户原始输入及用户主动插入的 `**` 加粗标记，不自动拆分、改写或重新组织内容。预览只将成对的 `**文字**` 安全渲染为粗体，不执行HTML。

# 简历定制 App 产品方案 V1.0

## 一、产品定位

这是一个固定模板的在线简历制作工具。

用户可以：

1. 按照固定字段填写个人简历内容。
2. 使用平台提供的提示词，前往免费AI大模型生成或优化简历内容。
3. 将生成后的内容粘贴回对应字段。
4. 实时预览白底黑字的标准简历。
5. 导出为PDF文件。

平台不调用AI API，不承担简历内容的生成、审核和判断，只提供内容填写工具、固定简历模板和辅助提示词。

## 二、核心产品原则

### 1. 内容完全由用户负责

平台不判断用户填写的内容是否：

* 太长
* 太短
* 缺乏数据
* 缺乏结果
* 表达空泛
* 与岗位不匹配
* 超过一页

平台不对用户填写内容进行限制、评分或修改。

### 2. 平台只提供辅助提示词

用户可以自行将提示词复制到：

* ChatGPT
* DeepSeek
* 豆包
* 通义千问
* Kimi
* 其他大模型平台

用户生成内容后，自行决定是否使用，并将最终内容粘贴回简历编辑器。

### 3. 固定模板，不提供自由排版

简历采用统一的白底黑字模板。

用户只能填写内容，不能修改：

* 字体
* 字号
* 颜色
* 模块位置
* 页面布局
* 分割线样式
* 简历整体结构

这样可以保证导出的简历格式稳定、整洁，适合招聘场景。

### 4. 不限制简历页数

简历可以自然生成一页、两页或更多页面。

平台不提示简历超过一页，也不强制用户压缩内容。

应届生、实习生、社会招聘用户都可以按照自己的实际经历决定简历长度。

# 三、页面结构

整个应用采用上下排列的单栏结构，不采用左右分栏。

页面从上到下依次为：

```text
顶部导航栏

简历内容填写区域

AI辅助提示词区域

简历实时预览区域

导出PDF按钮
```

## 1. 顶部导航栏

包含：

* 产品名称
* 保存状态
* 清空内容
* 导出PDF

例如：

```text
白简历 PlainCV

已自动保存              清空内容    导出PDF
```

## 2. 简历填写区域

采用纵向排列。

```text
个人信息
教育背景
实习经历
项目经历
个人优势
```

每个模块使用简单的白色表单区域，不使用复杂卡片、渐变或装饰。

## 3. AI辅助提示词区域

位于简历填写区域之后。

只提供四类提示词：

1. 实习经历优化
2. 项目经历优化
3. 根据JD定制实习经历和项目经历
4. 个人优势生成或优化

每个提示词提供：

* 提示词名称
* 简短使用说明
* 查看提示词
* 一键复制

## 4. 简历预览区域

位于页面下方。

使用白色纸张样式展示最终简历，按照从上到下的方式排版。

简历预览可以自然延伸为多页，不设置一页高度限制。

# 四、固定简历字段

## 1. 个人信息

只包含：

* 姓名
* 手机号
* 邮箱

不添加其他字段。

简历展示形式：

```text
张三

138xxxxxxxx ｜ example@email.com
```

---

## 2. 教育背景

字段只包含：

* 学校
* 专业
* 学历
* 入学时间
* 毕业时间
* 主修课程，选填
* GPA或专业排名，选填

允许用户添加多段教育经历。

简历展示形式：

```text
教育背景

齐鲁工业大学                    2024.09—2028.06
金融科技｜本科
主修课程：统计学、金融学、Python程序设计
专业排名：前20%
```

没有填写的选填字段不显示。

---

## 3. 实习经历

字段只包含：

* 公司名称
* 岗位名称
* 开始时间
* 结束时间
* 工作内容

允许用户添加多段实习经历。

工作内容使用多行文本框，由用户自由填写。

平台不限制：

* 字数
* 行数
* 内容结构
* 要点数量
* 表达方式

简历展示形式：

```text
实习经历

字节跳动        2024.12—2025.12
实习生

参与需求文档整理、功能测试、Excel数据处理和任务跟踪。
协助整理技术资料、异常样本和项目会议记录。
```

---

## 4. 项目经历

字段只包含：

* 项目名称
* 项目角色，选填
* 开始时间，选填
* 结束时间，选填
* 项目描述

允许用户添加多个项目。

项目描述使用多行文本框，由用户自由填写。

平台不限制：

* 字数
* 行数
* 描述结构
* 项目数量
* 内容表达方式

简历展示形式：

```text
项目经历

AI简历初筛工作流                2026.03—2026.05
项目角色：工作流开发与测试

使用Coze搭建简历解析、字段校验、岗位匹配和结果输出流程。
参与工作流测试、问题记录和后续迭代。
```

没有填写项目角色或时间时，对应内容不显示。

---

## 5. 个人优势

只有一个多行文本输入框。

用户可以自由填写：

* 一段文字
* 多段文字
* 多条内容

平台不限制内容格式和字数。

简历展示时按照用户输入的换行方式显示。

---

# 五、简历模板设计

## 1. 视觉风格

采用标准企业简历风格：

* 白色背景
* 黑色文字
* 深灰色辅助文字
* 从上到下排列
* 不使用侧边栏
* 不使用头像
* 不使用图标
* 不使用彩色模块
* 不使用技能进度条
* 不使用复杂图形
* 不使用双栏简历结构

## 2. 模块顺序

固定为：

```text
姓名
手机号和邮箱

教育背景

实习经历

项目经历

个人优势
```

用户不能调整模块顺序。

## 3. 多页处理

简历内容超过一页时自然分页。

需要避免：

* 一个模块标题单独出现在上一页底部
* 公司名称和工作内容被不合理拆开
* 项目名称和项目描述被不合理拆开

可以通过打印CSS中的分页规则尽量保证完整性。

不设置：

```css
height: 297mm;
overflow: hidden;
```

而是使用：

```css
width: 210mm;
min-height: 297mm;
```

让简历内容自然向下延伸。

# 六、AI辅助提示词功能

## 1. 实习经历优化提示词

用途：

用户提交自己的真实实习信息，让AI整理成适合简历展示的内容。

平台只提供提示词，不上传用户信息，不调用AI接口。

## 2. 项目经历优化提示词

用途：

用户提交项目背景、个人工作和项目结果，让AI整理项目描述。

## 3. 根据JD定制实习和项目经历提示词

用途：

用户提供：

* 目标岗位JD
* 原始实习经历
* 原始项目经历

AI在不编造经历的前提下，调整内容顺序、关键词和表达重点。

## 4. 个人优势提示词

用途：

用户提供：

* 求职方向
* 教育背景
* 实习经历
* 项目经历
* 技能情况

AI生成适合放在个人优势模块中的内容。

# 七、数据保存

第一版不需要用户注册，也不需要后端数据库。

用户填写的内容保存在当前浏览器中。

使用：

```text
localStorage
```

实现：

* 自动保存
* 页面刷新后恢复
* 关闭浏览器后再次打开仍可恢复
* 用户可以主动清空数据

清空数据前需要弹出确认提示。

---

# 八、PDF导出

第一版使用浏览器打印功能导出PDF。

用户点击“导出PDF”后执行：

```javascript
window.print();
```

打印时：

* 隐藏顶部导航栏
* 隐藏表单编辑区域
* 隐藏提示词区域
* 只显示简历内容
* 使用A4纸张宽度
* 支持自然分页
* 保留可以复制的文字

用户在浏览器打印窗口中选择：

```text
另存为PDF
```

# 九、技术方案

## 前端框架

使用：

```text
Next.js + React + TypeScript
```

它们分别负责：

* Next.js：搭建整个网页项目
* React：处理表单和实时预览
* TypeScript：减少字段和数据结构错误

## 表单处理

使用：

```text
React Hook Form
```

负责管理用户填写的内容。

## 基础数据校验

可以使用：

```text
Zod
```

但只进行必要的技术校验，例如：

* 数据是不是字符串
* 某个经历是否有唯一ID
* 本地数据格式是否损坏

不对简历内容进行质量判断。

不检查内容长短，不检查是否量化，不检查是否有结果。

## 数据保存

使用：

```text
localStorage
```

第一版不需要后端和数据库。

## 页面样式

使用：

```text
CSS Modules
```

第一版不需要引入复杂组件库。

## PDF导出

使用：

```text
浏览器打印功能 + CSS打印样式
```

第一版不使用收费PDF服务，也不将简历转换成图片。

# 十、第一版数据结构

```typescript
interface ResumeData {
  personalInfo: PersonalInfo;
  education: EducationItem[];
  internships: InternshipItem[];
  projects: ProjectItem[];
  advantages: string;
}

interface PersonalInfo {
  name: string;
  phone: string;
  email: string;
}

interface EducationItem {
  id: string;
  school: string;
  major: string;
  degree: string;
  startDate: string;
  endDate: string;
  courses?: string;
  gpaOrRanking?: string;
}

interface InternshipItem {
  id: string;
  company: string;
  role: string;
  startDate: string;
  endDate: string;
  content: string;
}

interface ProjectItem {
  id: string;
  name: string;
  projectRole?: string;
  startDate?: string;
  endDate?: string;
  description: string;
}
```

平台保存用户原始输入，不自动拆分、改写或重新组织内容。
